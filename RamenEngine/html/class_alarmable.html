Detailed Description

Allows GameObjects to call functions at a specific time.

All Commands must be Registered and Deregistered by the GameObject to be used correctly. All GameObjects have SubmitAlarmRegistration and SubmitAlarmDeregistration;

There are a total of 3 alarms (0-2) in each GameObject. TriggerAlarm is used when the amount of time given to registration is met.

◆ SubmitAlarmDeregistration()

void Alarmable::SubmitAlarmDeregistration ( AlarmableManager::ALARM_ID  id)
protected

Ask for Deregistration of a specific alarm for a specific time. Errors will occur if you try to deregister the same alarm more then once on the same GameObject.

Parameters
idThe specific alarm used.
Note
This is used when an object is being recycled and not destroyed. The alarm needs to be deregistered or it will still go off when the GameObject is already recycled. Not doing this can lead to unwanted results.

Example

void Bullet::SceneExit() //Bullet is still alive and being sent to a Factory
{
Updatable::SubmitUpdateDeregistration();
Drawable::SubmitDrawDeregistration();
Alarmable::SubmitAlarmDeregistration(AlarmableManager::ALARM_0);
BulletFactory::RecycleBullet(this);
}
