Update Notes

On (mostly) every large upload, notes of what the current version has added will be on this page.

3/9/2018

New Stuff:

  • AssetTool

    1. Getting Assets are even easier with a GameObject Tool. Instantly Get Models, Textures, Shaders, and Images by using this!

    GraphicObj = new GraphicsObject_WireframeConstantColor(AssetTool::Model("radModelName"), AssetTool::Shader("randomShader"), currentColor);

  • RamenAssets

    1. Another UserTool to allow Users instant access to Default Shaders and Models. GraphicsObj = new GraphicsObject_ColorNoTexture(AssetTool::Model(RamenAssets::Models::Axis()), AssetTool::Shader(RamenAssets::Shaders::ColorNoTextureConstant()));

    Not the prettiest looking but still good to remind Users.

Polished Shtuff

  • Faster deletion of SingleKeyInputs
    1. Inputables now have references to where key events are placed in the SingleKeyManager lists. Retrival and removal no longer needs to search inside the list.
    2. The list OF SingleKeyManagers in the KeyboardEventManager do not have a reference... yet.

3/6/2018

Coming Soon(ish):

  • EngineOptions
    1. Soon (if Andre thinks its a good idea) will be adding GameOptions. This will allow the user to decide what to do and dont want the engine to do. Example right now is the SpriteFactory. The Engine currently controls it but the user could turn it off if they want!

New Stuff:

  • Sprite
    1. Now Documented! (With what is currently available and working).

Polished Shtuff

  • Sprite

    1. Started Seperating Engine code from user code for GameOptions. RemoveAndRecycle is specifically for the SpriteFactory. For DISABLED SpriteFactory: Deleting the sprite will not call the factory for recycling.

    coolSprite->RemoveAndRecycle(); // used for the SpriteFactory

    or

    delete coolSprite; // Allows the user to delete the sprite if SpriteFactory isnt used.

3/4/2018

New Stuff:

  • Sprite
    1. Now added into the GameObject. Make sprites easier having to add the header everytime.
    2. To Use a Sprite Submit for a SpritableRegistration. Any Sprites rendering inside the new Spritable::Draw2D() function will be called to the screen. Sprites do not need to have a specific Camera to render to. The default is the new 2DCam!
    3. Sprites are Recycled after every scene to be used on other scenes. Hopefully not to waste space. At the moment Scenes Terminate Sprites when they are destroyed. Soon this will be in the SceneManager to avoid this.
    4. Sprites do not need to be deleted by the user. When you are done with the sprite, use Sprite::Destroy() and you are done!
  • SpriteTool
    1. Just like the CameraTool, it allows users to have use of Sprite Creation without needing the SpriteFactory.h
    2. SpriteTool is already in GameObject, just type Sprite tool and you're set!

Polished Shtuff

  • CameraManager
    1. Camera Manager has two currentCameras: CamManager::currentCam and CamManager::current2DCam. Both can be used at the same time for 3D objects and Sprites.
  • Attorneys
    1. Cleaned up Attorneys some for better access and readability.
