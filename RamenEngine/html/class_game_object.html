Detailed Description

The game entity that holds most information to see, control, and interact with. A GameObject is made up of multiple Components: Drawable, Updatable, Alarmable, Inputable, and Collidable. GameObjects are used within a Scene and can use any of these Components (depending if the user asks for submission). Specific functions like Update and Draw are GameObject specifically to allow the GameObject to be used per frame.

Example

#include "..\src\GameObject.h"
class TankObject : public GameObject
{
public:
TankObject();
virtual ~TankObject();
}

◆ SceneEntry()

void GameObject::SceneEntry ( )
privatevirtual

Can only be used if SubmitEntry has been used within a GameObject. This works well with Factories to be able to easily recycled. This is when the object wants to be connected to the Scene.

Example

void Bullet::SceneEntry()
{
Updatable::SubmitUpdateRegistration();
Drawable::SubmitDrawRegistration();
CollisionRegistration<Bullet>(this);
SetColliderModel(GraphicsObj->getModel());
}
