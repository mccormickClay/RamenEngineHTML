Detailed Description

Manager for SingleKeyManagers. This class holds the information of all SingleKeyManagers. When a AZUL_KEY wants to be managed it will be sent to a SingleKeyManager that focused only on that AZUL_KEY type.

Clay, 2/28/2018.

SingleKeyManager::KeyListRef &  pDeleteRef

Deregisters a GameObject as an Inputable to SingleKeyManager for a specific key and Key Event. If the key does not yet have a SingleKeyManager, it will lead to errors. If it is correctly to stored already. Then it is removed from the SingleKeyManager.

Clay, 2/28/2018.

Parameters
kThe key that is wanted to be registered.
evThe event (KEY_PRESSED, KEY_RELEASED) that is wanted to be registed.
pDeleteRefThe reference to where the specific key and event is located in the KeyList (either Pressed or Released). This allows for Quick Delete.
