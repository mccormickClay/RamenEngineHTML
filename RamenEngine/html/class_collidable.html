Detailed Description

Allows GameObjects to collide with one other by different Volumes.

All Commands must be Registered and Deregistered by the GameObject to be used correctly. All GameObjects have CollisionRegistration(C*c) and CollisionDeregistration(C*c).

For Collisions to be read in a specific way ( Scene::SetCollisionPair() or Scene::SetCollisionSelf() ) these functions must be used inside a Scene.

◆ SetColliderModel()

void Collidable::SetColliderModel ( Model *  pModel)
protected

Uses the information from a model to make a collision for an object. A GraphicsObject can be used to get the model information. Check GraphicsObject_Base for more information.

Note
This does not have to be the model from the Gameobject its being used in.
Parameters
pModelPointer to model that will be set as ColliderModel for GameObject.

Example

TankObject::TankObject()
{
SubmitUpdateRegistration();
GraphicsObj = new GraphicsObject_TextureLight(AssetManager::GetModel("t99body"),
AssetManager::GetShader("textureLight"), AssetManager::GetTexture("t99body"),
AssetManager::GetTexture("t99body"), AssetManager::GetTexture("t99track"), LightColor, LightPos);
SetColliderModel(GraphicsObj->getModel());
CollisionRegistration<TankObject>(this);
}
