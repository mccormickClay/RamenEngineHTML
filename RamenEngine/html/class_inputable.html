Detailed Description

Allows Gameobjects to use key commands on screen. This is seperate from Update(). This is more efficient by not asking every frame for when a keystoke was made. There are two events a key can have: KEY_PRESS AND KEY_RELEASE.

All Commands must be Registered and Deregistered by the GameObject to be used correctly. All GameObjects have SubmitKeyRegistration() and SubmitKeyDeregistration();

<...> // move Airplane forward
break;
case AZUL_KEY::KEY_SPACE:
<...> // fire bullet
break;
default:
DebugMsg::out("\nERROR: Key %i unprocessed", k);
}
